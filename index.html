JAVA SCRIPT

    Introduction:

  • JAVA SCRIPT : To recieve a request and provides response by applying some buisness logic.

  • Program: Set of instructios to do any task.

  • Application: Collection of programms to build any software.

  • From Frontend/UI we can recieve any request. Only static application (html,css).

  • Dynamic Applications : We can change by request (html,css,js) raised by user webpage. By using js we can make an application dynamic.

  • Standalone Application : works without any internet.

    Ex: Calculator , Gallery.

  • We can perform from validation using Js.

  • Dynamic : Application runs based on request raised model.

  • We need one processor/server for providing response based on request.

  • Server should maintain at Backend.

  • Js runs only with browser.

  • With help of Node.Js we can run Js anywhere i.e. (out of browser).

    Engines:

  1. Chrome - V8 ENGINE.

  2. Firefox - SPIDERMONKEY ENGINE.

  3. Edge - CHAKRA ENGINE (old version replaced now by V8).

  4. Safari - NITRO (Js core).

  • This engines are nothing but a piece of code.

  • V8 engine is developed by combo of C++,Js,Python,Type script.

  • Because of V8 engine in chrome we can run Js in chrome.

    3-Tier Architecture - 3 Phases

  1. html,css,js - Frontend presentation layer.

  2. Js - Backend buisness logic layer.

  3. Data base - User interface layer.

  4. Frontend & Backend are connected through API.

    Why Js ?

  • For making application dynamic.

  • For form validation.

  • Raising request from Frontend and providing response to Backend.

  • In Frontend - Js will act as a Scripting Language.

  • In Backend - Js will act as a Programming Language.

  • Node.Js will create a runtime environment for Js with the help of V8 engine.

  • Any application will work based on the request raised model.
